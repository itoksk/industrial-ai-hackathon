02

課題発見の手法（QC検定準拠）

なぜなぜ分析（5 Whys）

問題の根本原因を探るため、「なぜ？」を5回繰り返す手法。 トヨタ生産方式で有名な、シンプルだが強力な分析ツール。

  1. 問題を明確に定義する
  2. 「なぜその問題が起きたか？」を問う
  3. 答えに対してさらに「なぜ？」を問う
  4. 5回程度繰り返し、根本原因に到達
  5. 根本原因に対する対策を立案
例：不良品が多い なぜ1→検査が不十分 なぜ2→検査基準が曖昧 なぜ3→マニュアルが古い なぜ4→更新体制がない なぜ5→責任者が不明確

特性要因図（フィッシュボーン図）

問題の要因を体系的に整理する図。魚の骨のような形で、 4M（Man, Machine, Material, Method）などの観点から要因を分析。

  1. 解決したい問題を魚の頭に書く
  2. 大骨として主要因（4M）を配置
  3. 各主要因に対して小骨（詳細要因）を追加
  4. ブレインストーミングで要因を洗い出し
  5. 重要な要因を特定して対策検討

パレート図（80-20の法則）

問題の80%は20%の要因から生じるという法則に基づき、 重要な要因から優先的に対処するための分析手法。

  1. データを収集し、要因別に分類
  2. 頻度や影響度の大きい順に並べ替え
  3. 棒グラフと累積曲線を作成
  4. 累積80%までの要因を重点項目に
  5. 重点項目から優先的に改善

ブレインストーミング

チームで自由にアイデアを出し合う創造的問題解決法。 批判禁止、量重視、自由奔放、便乗歓迎の4原則。

  1. テーマを明確に設定
  2. 批判・評価は一切しない
  3. 質より量を重視してアイデア出し
  4. 他人のアイデアに便乗OK
  5. 後でアイデアを整理・評価
