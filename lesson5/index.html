📊 ヒストグラム（度数分布図）

ヒストグラムとは？
データの分布状態を視覚的に表現する棒グラフです。横軸にデータの範囲（階級）、縦軸に度数（データの個数）を取ります。

作成手順：

  1. データを収集する（最低30個以上が望ましい）
  2. データの最大値と最小値を確認する
  3. 階級数を決める（√データ数 または 5～20程度）
  4. 階級幅を計算する：(最大値-最小値)÷階級数
  5. 各階級に含まれるデータ数を数える
  6. 棒グラフを作成する（隙間なく並べる）

何がわかるか：

  • 中心傾向：データが集中している値（平均値付近）
  • ばらつき：データの散らばり具合（標準偏差の視覚化）
  • 分布の形：正規分布、偏り（歪度）、二山分布など
  • 異常値：離れた位置にある外れ値の存在
  • 工程能力：規格値との比較で品質管理状態を判断
💡 活用例：製品の寸法測定値100個をヒストグラムにすると、平均10.0mm、標準偏差0.1mmの正規分布が見られた。規格値9.8～10.2mmに対して工程能力は十分と判断。

📉 散布図（相関図）

散布図とは？
2つの変数の関係性を点の分布で表現するグラフです。横軸と縦軸にそれぞれ異なる変数を取り、対応するデータを点でプロットします。

作成手順：

  1. 2つの変数のペアデータを収集する（30組以上推奨）
  2. 横軸（原因系）と縦軸（結果系）の変数を決める
  3. 各軸の目盛りを設定する
  4. データを1つずつ点でプロットする
  5. 必要に応じて回帰直線を引く
  6. 相関係数を計算する（-1～+1の値）

何がわかるか：

  • 正の相関：右上がりの傾向（例：温度↑→反応速度↑）
  • 負の相関：右下がりの傾向（例：価格↑→販売数↓）
  • 無相関：ランダムな分布（変数間に関係なし）
  • 相関の強さ：点の集まり具合（|r|>0.7で強い相関）
  • 外れ値：傾向から外れた異常データの発見
  • 非線形関係：曲線的な関係性の存在
💡 活用例：加工時間と不良率の散布図を作成したところ、相関係数r=0.85の強い正の相関を発見。加工時間を短縮すると不良率が下がることが判明。

📈 管理図（Control Chart）

管理図とは？
時系列でデータをプロットし、工程が統計的管理状態にあるかを判断するグラフです。中心線（CL）と管理限界線（UCL/LCL）を設定します。

作成手順：

  1. 管理する特性値を決める（寸法、重量、不良率など）
  2. サンプリング方法を決める（頻度、個数）
  3. 20～25群のデータを収集する
  4. 平均値（中心線CL）を計算する
  5. 標準偏差を求め、管理限界線を設定する
    　UCL（上方管理限界）= 平均 + 3σ
    　LCL（下方管理限界）= 平均 - 3σ
  6. 時系列でデータをプロットする
  7. 異常パターンをチェックする

何がわかるか：

  • 工程の安定性：管理限界内にデータが収まっているか
  • 異常の検出：管理限界を超えるデータの発生
  • 傾向の把握：連続上昇/下降などのトレンド
  • 周期性：規則的な変動パターンの存在
  • 工程能力：規格値に対する余裕度（Cp, Cpk）
  • 改善効果：対策前後での変化の確認

異常判定ルール（シューハート管理図）：

  • 管理限界線を超える点がある
  • 9点が連続して中心線の同じ側にある
  • 6点が連続して増加または減少している
  • 14点が交互に上下している
💡 活用例：製品重量の管理図で、25日目から3点連続でUCLを超えた。調査の結果、材料供給装置の故障が判明し、即座に修理対応。

1. なぜなぜ分析の記入例

レベル 質問 回答例
なぜ1 なぜ納期が遅れるのか？ 生産が予定より遅れているから
なぜ2 なぜ生産が遅れるのか？ 機械の故障が頻発しているから
なぜ3 なぜ機械が故障するのか？ 定期メンテナンスが実施されていないから
なぜ4 なぜメンテナンスが実施されないのか？ メンテナンス計画が作成されていないから
なぜ5 なぜ計画が作成されないのか？ 担当者が明確でなく、責任の所在が曖昧だから

2. フィッシュボーン分析の要因整理例

【人 - Man】

  • 作業員の熟練度不足
  • シフト交代時の引継ぎ不備
  • メンテナンス担当者の不在

【機械 - Machine】

  • 老朽化による故障頻発
  • 予備部品の在庫不足
  • 定期点検の未実施

【材料 - Material】

  • 原材料の納入遅延
  • 品質不良による再加工
  • 在庫管理の不備

【方法 - Method】

  • 生産計画の精度不足
  • 作業手順書の更新遅れ
  • 緊急対応マニュアルの不備

遅延原因 発生件数 累積件数 累積比率 対策優先度

🚀 Google Apps Scriptとは？

GoogleのサービスをJavaScriptで自動化・拡張できるクラウドベースのプラットフォームです。

  • 無料で使える（Googleアカウントがあれば）
  • サーバー不要（Googleのインフラで動作）
  • スプレッドシート、ドキュメント、フォームと連携
  • Webアプリとして公開可能
  • 定期実行（トリガー）設定可能

  1. Google Driveを開く

  3. プロジェクト名を「品質管理アプリ」に変更

// 基本的なWebアプリケーション
function doGet() {
  const html = HtmlService.createHtmlOutputFromFile('index')
      .setTitle('品質管理ダッシュボード')
      .setXFrameOptionsMode(HtmlService.XFrameOptionsMode.ALLOWALL);
  return html;
}

// データを返すAPI
function getData() {
  return {
    message: "工場の品質データ",
    不良率: 3.5,
    生産数: 1000,
    更新日時: new Date().toLocaleString('ja-JP')
  };
}

  }));
  
  return paretoData;
}
